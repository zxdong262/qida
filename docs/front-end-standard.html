原则

统一的编码规范的目的是保持良好的阅读性，可继承性，规范性，以期望给开发带来便利。并不意味着本代码规范更正确或者更先进。

建议

  • 搜索和实验: 有时候搜索未必能找到合适的答案，幸运的是可以很方便的实验前端代码，比如通过 jsfiddle
  • 英文阅读：我确信通过良好的习惯和努力可以做到无障碍阅读英文文档，那就是从现在开始每次都阅读英文文档，遇到不认识的单词立刻查，最终有一天你会发现这次不用查了。
  • 重复的代码应该被抽象出来，写成统一的接口。
  • validate: typo(笔误，拼写错误，不匹配的标签）常常是百思不得其解的问题所在，css, html validate 非常有助于发现这些错误，同时应该在尽可能的情况下书写符合规范的代码，这有助于避免潜在问题。 css validate html validate

javascript 代码规范


      ///1. 缩进 ，4 个空格，不要用 tab
      ///2. 每个"," 后空一格
      ///3. 运算符前后都留有空格
      function add(a, b) { 
          return a + b 
      }

      ///4. 不要使用 “;” ,例外情况是 改行以"()" 或者 "[]"开头,如
      var a = b
      ;['apple','cake'].push('book')
      ;(a === b?a : b) = 15

      ///5. ',' 分割，每个一行时候， ','前置
      var a = {
          name: 'cake'
          ,age: 3
          ,sex: 'male'
      }

      var a = 14
      ,b = 12
      ,c = 3
      ,d = 34

      ///6. 元素较短时候，可以在一行
      var a, b, c

      ///7. 如果 'if(something)' 只有一行，请不使用 '{}'
      if(true) return a   //yes

      if(true) {return a} //no'

      if(true) return
      a                   //wrong  

      ///8. 仅使用 '' , 例外是在字符串内部或者 json格式
      var string = 'cake'
      ,a = ' div class="blue" '
